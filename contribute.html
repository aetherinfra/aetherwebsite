Contribution Workflow

  • Fork & Clone: Create a personal fork of the Aether WebOS repository and clone it to your local AArch64 development environment.
  • Feature Branch: Always create a new branch for your feature or bugfix (e.g., feat/virtio-net).
  • Atomic Commits: Ensure your commits are small, descriptive, and follow our naming conventions.
  • Pull Request: Submit your PR to the main branch. The Core Maintainers will review your code for architectural alignment.

Rules & Code of Conduct

To maintain a professional research environment, all contributors must adhere to:

  • Technical excellence and code safety.
  • Respectful communication in PR comments and Issues.
  • No plagiarism; all code must be original or appropriately licensed (GPL/MIT).
  • Maintain the "Bare Metal" philosophy—minimal dependencies.

Contributor Recognition

Certificate of Appreciation

Every successful contributor whose code is merged into the Aether ecosystem will receive a digital Certificate of Appreciation signed by the Core Maintainers. This certificate serves as official proof of your contribution to low-level systems engineering and can be used in professional portfolios and resumes.
